របៀបសរសេរ Git Commit

ទម្រង់: type: short description in lowercase

feat

មុខងារថ្មីៗ (New Features)

ប្រើពេលបន្ថែម UI components ថ្មី, logic ថ្មី ឬ screen ថ្មីៗ។

git commit -m "feat: add registration view screen" git commit -m "feat: implement login viewmodel with provider"
fix

ជួសជុល Bug (Bug Fixes)

ប្រើពេលជួសជុល App crash, layout ធ្លាយ ឬខុស logic។

git commit -m "fix: resolve null pointer crash on home screen" git commit -m "fix: update broken api endpoint url"
style

កែសម្រួល UI & Formatting

កែតម្រូវទម្រង់កូដ, padding, ពណ៌ ឬរៀបចំកូដឱ្យស្អាត (មិនប៉ះពាល់ logic ទេ)។

git commit -m "style: adjust spacing on job card component" git commit -m "style: format code with dart format"
refactor

រៀបចំកូដឡើងវិញ (Refactor)

កែសម្រួលរចនាសម្ព័ន្ធកូដឱ្យកាន់តែប្រសើរ និងស្អាតជាងមុន ដោយមិនបន្ថែមមុខងារថ្មី ឬជួសជុល bug ឡើយ។

git commit -m "refactor: clean up mvvm architecture structure" git commit -m "refactor: decouple repository data mapping logic"
chore / ci

ការងារទូទៅ & Pipelines

បន្ថែម package ផ្សេងៗ, កែប្រែ configuration នានា ឬរៀបចំ setup ឯកសារ pipeline។

git commit -m "chore: add provider package" git commit -m "ci: setup automated testing pipeline"
